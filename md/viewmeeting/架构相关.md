* 页面渲染
  * 远程调用架构：调用远程服务返回渲染页面，服务端计算压力大，客户端维护方便（服务升级时，一般仅服务端进行变化）
  * 端到端架构：服务返回数据，客户端本身进行页面渲染，服务计算压力小，客户端维护成本高（服务要变化时，要进行较多的处理保持节点一致）
  * 按照本身需求来进行选择

* 版本号：
  * 一般不通过传参来传递
  * 通过 url
  * 通过 http header
* 生产环境日志级别
  * 重要的日志信息 ，不能为 debug 级别（得不到，出不来）
  * 一般信息  Info
